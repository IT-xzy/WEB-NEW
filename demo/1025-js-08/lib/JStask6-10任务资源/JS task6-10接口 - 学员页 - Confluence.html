    JS task6-10接口

    转至元数据结尾
    • 创建： 李绍博，最新修改： 杨泽平 大约1分钟以前
    转至元数据起始

    说明：

    1. JS任务5-10统一采用萝卜多复盘项目接口
    2. 后台只有登录，无注册，注册在后台管理-角色管理中，任务5统一用提供的账密：admin   123456
    3. 学员环境：http://dev.admin.carrots.ptteng.com/;
    4. 跨域：
      学员服务器上的垮域已配置好，location拦截名为：
      后台——/carrots-admin-ajax/
      服务器nginx设置为：
      location /carrots-admin-ajax/{ 
      //更新时间：2017年8月11日 11:14:58
      proxy_pass http://175.25.23.192:10498/; 
      }
      （本地调试时可将该配置插入nginx配置文件中）
    5. 测试截图如下


     

    约定:

    1.查询多个的列表形式如id=2&id=3&id=4这种数据的形式。

    2.如无特殊标注，所有的Post和Put操作都以Application/json 的方式提交，这样适合于复杂的模型。 一些简单的操作应注明    “application/x-www-form-urlencoded”

    3.所有的返回字段都包含code和Message，code=0为正常，异常以 code为负表示，前端负责展示错误信息。

    4.Rest风格

    5./u/代表着是必须登录才可以访问的接口

     


    字段约定:

    字段 第几页 page 不写时默认为1

    字段 展示数量 size 不写时默认为10。

    article：

    字段 类别 type  0-首页banner 1-找职位banner  2-找精英banner 3-行业大图

    字段 来源 industry   type为3行业大图时使用 0-移动互联网 1-电子商务 2-企业服务 3-O2O 4-教育 5-金融 6-游戏

    字段 类别 status  1-草稿 2-上线
